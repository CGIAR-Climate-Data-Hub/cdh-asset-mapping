# CLAUDE.md — Working in cdh-asset-mapping

This file orients Claude Code when working in this repo. Read it once on entry.

## What this repo is

Reproducible pipeline for CGIAR Climate Data Hub (CDH) system-wide climate data asset mapping exercise. Centre submissions (Excel) are ingested, normalised, and used to generate figures plus report outputs. Primary authored report source is `report.qmd`. No statistics are hardcoded — everything derives from `data/normalized/assets.json`.

## Running the pipeline

```bash
pip install -r requirements.txt
.venv/bin/python src/ingest.py
.venv/bin/python src/figures.py
env QUARTO_PYTHON=.venv/bin/python quarto render report.qmd --to html
.venv/bin/python src/report.py
Rscript -e 'shiny::runApp("dashboard")'
python src/export_dashboard_data.py
```

## Key data conventions

- **Centre names**: match filenames in `data/submissions/` exactly (e.g. `CIFOR_ICRAF`, not `CIFOR-ICRAF`).
- **Hub-funded centres**: Alliance, IITA, ILRI, IFPRI, IWMI, WorldFish — defined in `HUB_FUNDED` set in `ingest.py`.
- **Domain vocabulary**: Hazard, Hazard / Climate Services, Exposure, Sensitivity, Adaptive Capacity, Adaptation Analytics, Mitigation, Multi-domain, Climate Policy / Finance, and hybrid labels. Defined in `DOMAIN_NORM` and `HYBRID_PATTERNS` in `ingest.py`. Do not add new top-level domains without agreeing the vocabulary with the CDH team.
- **Geo groups**: Africa, Global, Asia / South & SE Asia, Latin America & Caribbean, Multi-regional, Not specified. Defined in `GEO_NORM_MAP` in `ingest.py`.
- **Sheet join**: sheets are joined by row index (not by asset name), because names diverge slightly across sheets within the same workbook.

## Rules for Claude Code in this repo

1. **Never hand-edit `data/normalized/assets.json`.** It is a generated file. If an asset needs correction, fix the source Excel or the normalisation maps in `ingest.py`, then regenerate.
2. **Never hardcode statistics** in report outputs. Every number must be computed from data. Use shared helpers in `src/report_common.py`.
3. **Keep external links explicit and configurable** in report source. Set live dashboard/portal URLs near top of `report.qmd` rather than burying them in generated text.
4. **Update `merge_log.json` for consolidations** — do not silently remove duplicate assets from the JSON. The merge log is the audit trail.
5. **Adding a new centre**: add the filename to `CENTRE_FILES` in `ingest.py`, mark `HUB_FUNDED` if applicable, drop the Excel into `data/submissions/`, re-run the pipeline.
6. **`outputs/` is git-ignored** — commit figures and the report only when doing a deliberate release (move them out of `outputs/` or remove from `.gitignore` for that commit).
7. **Do not modify submission Excels** — they are source of truth from the centres. Flag data quality issues in comments or in a `data/quality_notes.md` file instead.

## Common tasks

**Re-run after new submission:**
```bash
# Drop new Excel into data/submissions/Centre.xlsx
# Edit CENTRE_FILES in src/ingest.py to add the centre
.venv/bin/python src/ingest.py
.venv/bin/python src/figures.py
env QUARTO_PYTHON=.venv/bin/python quarto render report.qmd --to html
.venv/bin/python src/report.py
```

**Update a normalisation rule:**
Edit the relevant map in `src/ingest.py` (`DOMAIN_NORM`, `GEO_NORM_MAP`, `TYPE_NORM`), then re-run `ingest.py` and downstream steps.

**Apply a new consolidation:**
Add an entry to `data/merge_log.json` under `"applied"` with the exact asset names as they appear in the Excel (Sheet A column 0), then re-run `ingest.py`.
